# Phase 2: シナリオ抽出プロンプト

Phase 1 で把握した画面構造を元に、E2E テストの対象シナリオを抽出してください。

実施事項:

1. ハッピーパスのシナリオを 1 つ書き出す（最も典型的な利用フロー）
2. エラーケースのシナリオを 2 件以上書き出す（入力ミス・必須項目漏れ・通信失敗など）
3. 境界値・特殊条件のシナリオを 1 件書き出す（最大文字数・空文字・連続クリックなど）

各シナリオの構造:

- 名前（命名規則: `<画面名>_<想定ユーザー行動>_<期待結果>`）
- 前提条件（auth 状態 / 事前データ / 直前画面）
- 操作手順（番号付き）
- 期待される最終状態（accessibility tree 上で何が見えるか）
- 観測すべき副作用（console エラー / API リクエスト）

参考ツール:

- `browser_console_messages` で `level: 'error'` のメッセージを採取する
- `browser_network_requests` で `/api/.*` などのフィルターをかけてレスポンスを観察する

注意:

- シナリオは独立して実行可能であること（前のシナリオの結果に依存しない）
- 「クリックする」と書く代わりに「<role> "<accessible name>" をクリックする」と書く
