# セルフレビュー指示

Phase 4 で生成したテストコードを、Phase 3 までとは別のセッション（推奨: Claude Code の sub-agents）で読み直してください。

このプロンプトは、コードを書いた本人ではなく独立した第三者目線で読むためのチェックリストです。

## レビュー観点

1. **シナリオの独立性**: テスト間で副作用が漏れていないか
2. **ロケータの安定度**: `getByRole` の `name` が UI 文言の微変更で壊れないか
3. **待機戦略**: `browser_wait_for` で書いていた箇所が `expect.toBeVisible` 等に置き換わっているか
4. **アサーションの粒度**: 「画面遷移完了」を 1 つのアサートで見ているか、過剰に細かく分割していないか
5. **副作用検証**: Phase 3 で観察した console.error / API ステータスを assert に反映しているか
6. **storage state の取り扱い**: 認証が必要なテストで適切に storage state がロードされているか
7. **重複の見落とし**: 同じシナリオが既存テストに既に存在しないか

## 出力フォーマット

```
## 結果: PASS | NEEDS_REVISION

### Tier 1（必ず修正）
- <ファイル:行番号> <問題> / 修正案

### Tier 2（推奨修正）
- ...

### Tier 3（任意）
- ...

### 確認したファイル
- tests/xxx.spec.ts
- ...
```

## 修正方針の判断

- Tier 1 が 1 件でもあれば Phase 4 をやり直す
- Tier 2 以下のみなら、修正案を採用するかは執筆者が判断する

## 注意

- レビュー時にテストを実行しない（実行は別工程）
- 「動かなさそう」を Tier 1 にしない。実行で判定する
- レビュー中に推測で修正コードを書かない（指摘のみ）
